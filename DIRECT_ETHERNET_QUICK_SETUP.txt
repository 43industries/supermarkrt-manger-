STEP-BY-STEP SETUP:
───────────────────────────────────────────────────────────

STEP 1: Configure Machine 1 (Server)
  → Right-click: SETUP_DIRECT_ETHERNET.bat
  → Select: "Run as administrator"
  → Follow prompts

  OR manually:
  1. Windows Settings → Network & Internet → Ethernet
  2. Click on your Ethernet adapter
  3. Click "Edit" under "IP assignment"
  4. Select "Manual"
  5. Enter:
     IP: 192.168.1.100
     Subnet: 255.255.255.0
  6. Click "Save"

STEP 2: Configure Machine 2 (Client)
  1. Windows Settings → Network & Internet → Ethernet
  2. Click on your Ethernet adapter
  3. Click "Edit" under "IP assignment"
  4. Select "Manual"
  5. Enter:
     IP: 192.168.1.101
     Subnet: 255.255.255.0
  6. Click "Save"

STEP 3: Connect Ethernet Cable
  → Connect cable between the two machines
  → Wait a few seconds for connection to establish

STEP 4: Test Connection
  On Machine 2, open Command Prompt:
  → Type: ping 192.168.1.100
  → Should see "Reply from 192.168.1.100"
  → If successful, connection works! ✅

STEP 5: Start Server on Machine 1
  → Double-click: START_OPTIMIZED.bat
  → Server starts on: http://192.168.1.100:3000
  → Keep server window open!

STEP 6: Connect from Machine 2
  → Open web browser
  → Go to: http://192.168.1.100:3000/app.html
  → Login: admin / admin123

TROUBLESHOOTING:
───────────────────────────────────────────────────────────

Ping fails?
  → Check cable is connected on both ends
  → Verify IP addresses are correct
  → Ensure both in same subnet (192.168.1.x)
  → Try restarting both machines

"No network access" icon?
  → This is normal for direct connections
  → Connection will still work
  → Can ignore this warning

Can't reach server in browser?
  → Verify server is running on Machine 1
  → Check firewall allows port 3000
  → Test: http://localhost:3000/app.html on Machine 1
  → Verify using correct IP: 192.168.1.100
